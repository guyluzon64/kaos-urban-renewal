# בניין, מה נסגר?

מיזם ציבורי ללא מטרות רווח שמרכז מידע רשמי על התחדשות עירונית ומציג אותו
בשפה פשוטה לתושבים. המערכת אינה נותנת ייעוץ משפטי, תכנוני, שמאי או נדל״ני,
ואינה חוזה אם תכנית או פרויקט יתקדמו.

## עדכון הנתונים

משורש הפרויקט:

```powershell
pip install -r requirements.txt
python scripts/update_public_data.py
```

הריצה:

1. מחפשת מקורות רשמיים דרך CKAN של `data.gov.il`.
2. מורידה משאבים ציבוריים נתמכים ושומרת אותם ב־`data/raw/`.
3. מתעדת כל מקור, הצלחה ושגיאה ב־`data/metadata/`.
4. יוצרת את קובצי העיבוד הבאים:

   - `data/processed/urban_renewal_raw_collected.csv`
   - `data/processed/urban_renewal_standardized.csv`
   - `data/processed/urban_renewal_public.csv`

5. כותבת דוח רענון ל־`data/metadata/latest_refresh_report.csv`.

שגיאה במקור אחד אינה מפסיקה את כל הריצה. אם אף מקור חי לא החזיר רשומות,
הסקריפט שומר את הנתונים הציבוריים הקודמים ומסמן זאת בדוח.

## הרצת האתר

```powershell
streamlit run app/urban_renewal_legal_scout.py
```

האתר קורא רק את `data/processed/urban_renewal_public.csv`; אין צורך בהעלאה
ידנית.

## מקורות

- API ציבורי של `data.gov.il`, כולל חיפוש CKAN ו־DataStore עם דפדוף.
- מאגר מתחמי ההתחדשות העירונית של משרד הבינוי והשיכון.
- שכבת מתחמי ההתחדשות העירונית המוכרזים, כאשר קיים משאב ציבורי יציב.
- רשימות יישובים ורחובות מ־`data.gov.il` לצורכי ניקוי וחיפוש בלבד.
- XPLAN, רק כאשר מזוהה נקודת ArcGIS ציבורית ויציבה.
- GovMap ומבא״ת כקישורי עזר רשמיים; האתר החזותי שלהם אינו נסרק.

כל רשומה שומרת, כאשר הם זמינים, את שם המקור, כתובת המקור, סוג המקור ומועד
העדכון.

## השלב הציבורי המזוהה

הקובץ הציבורי כולל מדד תיאורי של 0–9:

`שלב ציבורי מזוהה: X מתוך 9`

המדד מתאר רק מה ניתן לזהות ברשומות ציבוריות. הוא אינו הסתברות, המלצה,
קביעה משפטית, דירוג השקעה או הבטחה לגבי המשך התהליך.

## אוטומציה

הקובץ `.github/workflows/update_public_data.yml` מריץ רענון שבועי וגם מאפשר
הרצה ידנית. כאשר הנתונים משתנים, הוא מתחייב לקבצים המעודכנים באמצעות
הודעת הבוט `Update public urban renewal data`.
